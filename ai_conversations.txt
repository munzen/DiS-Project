The error is cur.fetchone() returning None on the query SELECT balance, sim_date FROM users WHERE user_id = %s. This means no user row was found for the session's user_id.

This is likely because:

The schema was changed (added balance column) but the DB wasn't re-initialized
OR the user registered before balance was added, so the column doesn't exist in the DB yet
OR the user doesn't exist in the DB (session has a stale user_id)
The most likely cause: the schema was updated with balance but the actual database table hasn't been updated.

Actually, fetchone() returning None means the query found no matching rows, not that the column is missing—that would throw a different error. So the real issue is the session contains a user_id that doesn't exist in the database anymore, probably because the DB was wiped when re-running the schema. The fix is to handle this gracefully by clearing the session and redirecting to login when fetchone() returns None.

fetchone() returns None — user_id in session doesn't exist in DB (likely DB was wiped when schema was re-applied). Fix: clear stale session and redirect to login.
